# Workflow агента

Три основных рабочих потока. Агент сам определяет, какой применять, по входной задаче.

## Поток 1. Стандартный (от задачи до промта)

**Триггер:** «Сделай промт под [модель] для [тип контента]»

1. Агент определяет: тип, модель, длительность/формат, платформу, цель.
2. Уточняет ТОЛЬКО критическое (продукт, ЦА, оффер, референсы — если их не хватает для качества).
3. Прогоняет через 4 роли последовательно.
4. Формирует JSON-промт под выбранную модель.
5. Прогоняет через чек-лист 13 пунктов.
6. Выдаёт:
   - Основной JSON в code-блоке
   - Negative prompt отдельно
   - Краткий комментарий (1–3 предложения)
7. По запросу — short version (≤1500 символов).

---

## Поток 2. Расширенный (с референсами)

**Триггер:** Прикреплены референсы персонажа / продукта / стиля.

1. Получаешь референс персонажа (Nano Banana output, фото, GPT Image 2 output).
2. Получаешь референс продукта.
3. Указываешь модель + длительность + формат + платформу.
4. Агент анализирует референсы и фиксирует **якоря консистентности**:
   - Персонаж: возраст, типаж, черты лица, причёска, особенности
   - Продукт: форма, цвет, материал, ключевая деталь
   - Стиль: цветовая палитра, свет, атмосфера
5. Агент строит промт с явной привязкой к референсам.
6. Выдача стандартная + блок `consistency_anchors` с описанием того, что должно остаться неизменным между кадрами/сценами.

---

## Поток 3. Audit (разбор слабого промта)

**Триггер:** «Разбери этот промт» / «Усиль промт» / «Ролик не работает».

1. Получаешь существующий промт (опционально — ссылка на результат).
2. Прогоняешь по чек-листу 13 пунктов, помечаешь каждый ✓/❌ с комментарием.
3. Описываешь коммерческие потери — что теряется в продажах из-за слабых мест.
4. Выдаёшь усиленную версию промта в JSON.
5. Объясняешь что главное было исправлено и какой эффект ожидается на метрику (CTR / удержание / конверсия).

См. шаблон `templates/prompt-audit.md`.

---

## Поток 4. Серия (несколько роликов с одной героиней / одним продуктом)

**Триггер:** «Серия из N роликов с одной героиней» / «3 видео для линейки продуктов».

1. Зафиксировать персонажа/продукт через Nano Banana или GPT Image 2 (если ещё нет референса).
2. Создать «мастер-блок» `consistency_anchors` — единый для всей серии.
3. Под каждый ролик создавать промт, копируя в него `consistency_anchors`.
4. Варьировать только сценарную часть (drama, action, emotion), сохраняя визуальные якоря.

## Связи с другими скиллами

Когда агент должен подгрузить существующий скилл:

| Ситуация | Скилл |
|----------|-------|
| Нужна сильная драматургия и сценарий | promo-video-director |
| Финальная оптимизация текста промта | prompt-master |
| Voice-over или субтитры в видео | humanizer |
| Видео делается под карточку Wildberries | wb-infographic-builder + wildberries-ai-operator |
| Клиент новый, нужна квалификация | ai-video-sales |
| После промта — оценить стоимость для клиента | ai-video-pricing |

---

## Что агент НЕ делает

- Не пишет промты под Midjourney / Runway / Kling без явного запроса с обоснованием
- Не выдаёт промт без negative prompt
- Не выдаёт промт без code-блока
- Не превышает 3500 символов без сегментации
- Не молчит при слабой задаче — сначала возражает, потом усиливает
- Не работает с 18+, чужими брендами без прав, лицами реальных публичных персон
